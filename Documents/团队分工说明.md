# 锄大地安卓项目团队分工说明（4人）

## 一、分工目标
结合《uml安卓游戏设计》与《锄大地游戏规则设计》，采用 MVC + 模块化协作方式，保证 5 周内可交付：
- 基础可玩的锄大地游戏流程
- 多人蓝牙联机框架
- 至少两种 AI 出牌策略
- UML 与实训文档材料

## 二、成员分工

### 成员 A（组长/项目管理）
- 负责内容：需求管理、迭代计划、仓库规范、集成发布
- 主要任务：
1. 整理需求与验收标准（规则、联机、AI、南北规则开关）
2. 管理 Sprint 看板与每周里程碑
3. 维护主分支合并策略、代码规范、版本标签
4. 组织联调和阶段性演示
- 交付物：
1. 需求说明与迭代计划文档
2. 项目周报与风险清单
3. 发布说明（Release Note）

### 成员 B（核心逻辑开发）
- 负责内容：Model + Rule + Controller
- 主要任务：
1. 实现牌数据结构（Card/Deck/Player/GameState）
2. 实现牌型识别（单张、对子、三条、炸弹、顺子、同花五、葫芦、铁支、同花顺）
3. 实现出牌校验、回合推进、首轮方块3、过牌规则、结算计分
4. 输出可测试的核心服务接口，供 UI 和联机层调用
- 交付物：
1. 核心规则引擎代码
2. 单元测试用例（规则判断与计分）
3. 核心模块设计说明

### 成员 C（安卓 UI 开发）
- 负责内容：View 层与交互流程
- 主要任务：
1. 完成页面：主页、房间页、对局页、结算页
2. 完成页面跳转与基础动画反馈
3. 对接 Controller 接口，完成选牌、出牌、过牌、提示
4. 适配手机分辨率，确保主要机型可用
- 交付物：
1. UI 页面与布局文件
2. 页面交互流程图
3. UI 测试记录（关键流程截图）

### 成员 D（蓝牙联机 + AI）
- 负责内容：Network + Strategy
- 主要任务：
1. 实现主机/客户端蓝牙房间连接流程
2. 实现对局消息协议（出牌、过牌、同步、结算广播）
3. 实现两种 AI 策略（如贪心、保守）并支持策略切换
4. 处理掉线重连和弱网（蓝牙不稳定）下的基本恢复
- 交付物：
1. 蓝牙通信模块
2. AI 策略模块
3. 联机稳定性测试记录

## 三、协作机制
1. 分支规范：`main`（稳定）/`dev`（集成）/`feature/*`（个人功能）
2. 提交流程：个人功能完成后发起 PR，至少 1 人 Review 后合并
3. 同步频率：每天站会 10-15 分钟，每周一次迭代回顾
4. 缺陷处理：按 P0/P1/P2 优先级分配，P0 当天修复

## 四、代码所有权建议（避免冲突）
1. 成员 A：`README.md`、`Documents/*`、项目管理配置文件
2. 成员 B：`app/src/main/java/com/scut/chudadi/model`、`rule`、`controller`
3. 成员 C：`app/src/main/res`、`MainActivity`、后续页面 Activity/Fragment
4. 成员 D：`app/src/main/java/com/scut/chudadi/network`、`ai`、联机相关 UI 同步
5. 测试文件：核心逻辑测试由成员 B 负责，联机与 UI 测试由成员 C/D 共同补齐

## 五、建议迭代节奏（5周）
1. 第1周：需求/UML定稿 + 项目骨架 + 基础UI原型
2. 第2周：核心规则引擎完成 + 本地单机流程打通
3. 第3周：蓝牙房间联机与消息同步初版
4. 第4周：AI策略接入 + 联调 + 缺陷修复
5. 第5周：压力测试、文档整理、最终答辩材料提交
